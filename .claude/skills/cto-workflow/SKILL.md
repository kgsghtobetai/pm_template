# CTO Workflow

## 概要
このスキルは、CTOとしての標準的な開発ワークフローを定義する。

## ワークフロー順序（厳守）

```
/exploration-phase → /create-plan → 実装 → /peer-review → /update-docs
```

## フェーズ定義

| フェーズ | 担当エージェント | 目的 |
|---------|------------------|------|
| 探索 | Investigation Agent | 仕様の穴を事前に特定し、明確化質問を行う |
| 計画 | Architect | 実装前に修正範囲と手順を明確化する |
| 実装 | Developer | プランに沿ったブレのないコードを出力する |
| 批評 | Critical Peer | 複数視点でレビューし品質を担保する |
| 蓄積 | Librarian | 失敗を教訓としてドキュメント化する |

## 使用方法

1. **探索フェーズ**: `/exploration-phase` を実行し、要件の穴を特定
2. **計画フェーズ**: `/create-plan` で詳細な実装計画を作成
3. **実装フェーズ**: 計画に沿ってコードを実装
4. **レビューフェーズ**: `/peer-review` で批判的視点からレビュー
5. **蓄積フェーズ**: `/update-docs` で教訓をドキュメント化

## 原則

- 各フェーズをスキップしないこと
- 前のフェーズが完了してから次に進むこと
- 問題が発見された場合は前のフェーズに戻ることを恐れないこと
